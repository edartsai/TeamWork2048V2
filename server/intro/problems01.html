1.選擇協作平台

✓ 平台 優點 缺點
Github
  • 可直接 publish webpage to everyone
  • 不清楚協作方式
Team Services+Azure
  • 熟悉介面
  • 版控思維類似SVN
  • 整合VS2015
  • 微軟的 Total Solution
    可一鍵完成 commit, build, test, publish
  • Team Services 本身無法 publish 成公開網頁
    需搭配 Azure
  • Azure功能太多、設定不易
  • Azure要錢....
Team Services+IIS
  • 熟悉介面
  • 版控思維類似SVN
  • 整合VS2015
  • 直接使用 local 檔案系統，publish快
  • 會議室連不到 IIS...
✓ Github
  • 協作方便
  • 可直接建立公開網頁
  • Github desktop GUI
  • push-code to webpage 有時間差
